# 30/60/90 Day Plan

## Purpose

This plan turns the repository into an execution program with near-term, medium-term, and structural outcomes.

## First 30 Days

### Objectives

- stabilize trust paths
- remove immediate credential ambiguity
- establish architectural visibility

### Deliverables

- complete current-state architecture map
- complete credential inventory with owners and storage locations
- define break-glass and recovery-material policy
- document exposed and shared credentials that require rotation
- define target identity model for Vault, Argo CD, Git access, and AI systems

## Days 31-60

### Objectives

- design and approve the future-state identity model
- reduce reliance on static credentials
- define the AI security governance baseline

### Deliverables

- ADR for Vault human access via OIDC
- ADR for Argo CD human access via OIDC
- ADR for GitHub App based machine access for Argo CD
- AI security and governance standard
- trust-boundary review for OpenClaw host-control and agentic flows

## Days 61-90

### Objectives

- begin implementation planning for the approved model
- validate recovery and operational resilience

### Deliverables

- implementation plan for Vault OIDC
- implementation plan for Argo CD OIDC
- implementation plan for GitHub App migration
- architecture review checklist for platform and AI changes
- AI system inventory with risk ratings
